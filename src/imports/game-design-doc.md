一、核心定位
游戏名称（暂定）
《我是一个“建”人》
类型
- 纯文字交互
- 数值驱动叙事（类似养成+随机事件）
- 多结局系统
- 时间推进制（以“学期”为单位）

游戏设定：
玩家是一名建筑学硕士研究生，面对建筑行业下行，决定转行互联网或外企。

请实现以下系统：

一、角色初始化系统
- 随机生成本科院校（QS10/清北、QS100/老八校、211、双非）
- 硕士院校层次不会比本科低，40%概率高一档、20%概率高两档、20%概率同层次、但20%概率同等层次。上限为
- 有20%概率为海外留学背景
- 显示学校名称与背景标签
  
二、基础属性系统
请实现以下属性（0-100）：
- 建筑专业力
- 逻辑力
- 表达力
- 英语能力
- 结构化思维
- 抗压值
- 人脉值
- 金钱
- 自我怀疑
- 年龄焦虑
  
属性变化要可视化展示（数值条）

三、时间系统
- 以“学期”为单位
- 总计6个学期（研一-研三）
- 每学期4回合
- 显示当前学期
  
四、行动选项（每回合可选一项）
- 改图
- 学产品
- 投实习
- 参加校招
- 准备雅思
- 做副业
- 摆烂
  
每个选项都会影响不同数值

五、公司系统
请建立公司难度等级：

互联网大厂：
腾讯、字节跳动、阿里巴巴、京东、百度、快手

外企科技：
Google、Microsoft、Amazon、Meta、Apple

咨询公司：
McKinsey、BCG、Bain

中厂：

传统路径：

不同公司有不同数值门槛。
